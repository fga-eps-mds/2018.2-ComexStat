Comex Stat API

1. Sobre a API

Essa é uma API dedicada a prover acesso aos dados relacionados ao comércio exterior de bens, provenientes do Ministério da Indústria, Comércio Exterior e Serviços (MDIC). A API é feita em GraphQL, cuja ferramenta de busca a banco de dados oferece recursos que facilitam as buscas e filtros nas pesquisas

2. Como usar

A estrutura de consultas na API é constituida de uma string em formatada em JSON contendo a operação a ser feita (que no caso dessa API sempre será um query), o método de busca a ser utilizado, com seus respectivos filtros, e os campos pesquisados. Como no exemplo a seguir:

No exemplo acima é primeiro definida a operação "query", dentro da qual é utilizado o método de busca "allImport" com um filtro que filtra pelo nome do país de origem (os filtros ficam entre parênteses), e o único campo requerido é o "ncm", que por sua vez tem um campo requerido chamado "countryNamePt". Há ainda, separando o método de busca e os campos, uma estrutura com "edges" e "node". Essas são apenas obrigatoriedades estruturais da linguagem que definem uma melhor organização dos resultados que serão mostrados. Em seguida, basta executar o query através do botão "Play" na API e os resultados serão retornados em formato JSON no container da direita

3. Métodos de buscas

A seguir estão descritos todos os possíveis métodos de busca que podem ser utilizados no query e seus respectivos valores de retorno. Bem como os filtros cabíveis a cada método.

3.1. allImport

Buscar por todas as importações

3.1.1. Filtros disponíveis

Para filtragem, estão disponíveis como parâmetro de consulta todos os campos da model (como por exemplo: "date", "countryNamePt", "ncmCode", "urfName", etc).

No caso do allImport também temos um filtro costumizado chamado "commercializedBetween", que filtra de acordo com o intervalo de datas. Podemos utilizá-lo no seguinte formato:

commercializedBetween:"[\"yyyy-mm-dd\",\"yyyy-mm-dd\"]"


A primeira data é o começo da busca e a segunda o fim da busca, obrigatoriamente no formato ano-mês-dia.

5. Documentação Dinâmica

O GraphiQL oferece também uma documentação dinâmica, que pode ser acessada pelo botão "Docs" no canto superior direito da API. Por lá podem ser vistas informações de todas as operações possíveis, incluindo descrições do funcionamento e listagem dos campos e filtros. Tudo dentro da própria API !

Além disso, outro recurso oferecido pelo GraphiQL para facilitar a interação do usuário com a API é a disponibilidade do recurso de autocomplete para todos os métodos de pesquisa e campos. Basta apertar as teclas Ctrl + Space e uma lista com as possibilidades será apresentada

6. Outros recursos

Além do que já foi apresentado, alguns outros recursos que a API oferece podem ser acessados pelos botões "Prettify" e "History" no canto superior esquerdo, o primeiro serve para automaticamente formatar o query de entrada em um JSON com uma boa legibilidade, e o segundo serve para ver o histórico de operações realizadas. Outra funcionalidade que deve ser citada são os permalinks, ao executar um query a URL ganha um parâmetro contento o JSON da operação feita, de tal forma que essa URL pode ser compartilhada para que outros usuários acessem diretamente esse mesmo query.
